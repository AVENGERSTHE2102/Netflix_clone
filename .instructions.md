# Project Directives & Flow Tracking

## Core Application Flow
1. **Signup Page**: The entry point. No Sign In page allowed.
2. **Video Intro**: Plays immediately after successful signup (acts as transition/loading).
3. **Who's Watching**: Profile selection screen shown after the video intro.
4. **Browse Page**: The main movie/content area shown after a profile is selected.

## Branding & UI Rules
- **Logo**: Use `/images/logo.png` (Complix branding). 
- **Logo Placement (Signup)**: Embed directly in the page body, above the form (no separate header frame).
- **Logo Size (Signup)**: Large (`220px` height).
- **Header**: No header button (Sign In/Up) in the top right.
- **Background**: Clear local storage only when at the root (`/`) to allow the Signup flow to start fresh, but preserve session for Browse.
- **Redirects**: Any unauthorized access must redirect to the root (`/`), never to a Sign In page.
- **Top 10 Numbers**: Must use a solid background fill (`#141414`) and `paint-order: fill stroke` to hide internal line intersections and keep a clean outer outline.
- **Validation**: No regex or strict restrictions on the signup form inputs.

## Technical Requirements
- **Local Auth**: Credentials must be saved to `localStorage`.
- **Auto-Logout**: `localStorage` must be cleared on every page refresh to ensure the landing/signup page is always shown first.
- **Video Assets**: Use WebM versions (`/videos/hori.webm`, `/videos/verti.webm`) for faster loading.
- **Loading UI**: Only a simple spinner if needed; remove complex "Netflix Intro" CSS animations.
